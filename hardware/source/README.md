# Editable hardware source release gate

This directory is reserved for the preferred modifiable source of Thin-Pod Gateway rev 0.1.

## Required release contents

The OSHWA-facing release must contain the exact KiCad project used to generate the final fabrication package, including at least:

```text
Thin-Pod_Gateway_rev0.1.kicad_pro
Thin-Pod_Gateway_rev0.1.kicad_sch
Thin-Pod_Gateway_rev0.1.kicad_pcb
fp-lib-table
sym-lib-table
footprints/
symbols/        # when project-local symbols are required
```

Names may differ from this convention, but the project must open without relying on private local paths or unavailable personal libraries.

## Current status

The editable KiCad project is not present in the repository as of the OSHWA preparation audit dated 13 July 2026. The fabrication package under `hardware/fabrication/` was generated by KiCad Pcbnew 10.0.1 on 21 May 2026 and records the project revision as `rev?`.

This README does not satisfy the source requirement. It makes the missing-source condition explicit so that repository documentation cannot imply that a complete modifiable hardware release already exists.

## Acceptance procedure

Before an OSHWA application is submitted:

1. copy the exact native design source into this directory;
2. remove absolute/private library references and include every required project-local dependency;
3. record symbol and footprint provenance;
4. open the project in a clean KiCad environment;
5. run schematic ERC and PCB DRC;
6. regenerate Gerber and drill outputs;
7. compare the regenerated outputs with the documented manufactured `rev 0.1f` package, explaining any intentional difference;
8. update the BOM from the final schematic and assembly record;
9. remove this open-gate wording only after the evidence is committed.

The editable source is the normative design. Gerbers and drill files are derived convenience outputs.
